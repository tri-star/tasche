# Merge Judgement

## 目的

AI review と人間 review の最終判定基準を統一し、Pull Request を止めるべきケースを限定する。

最終判定は次の 2 つのみとする。

- `mergeable`
- `human-review`

## 基本方針

この review の目的は、コード品質全般の採点ではない。
この PR を今すぐ安全にマージしてよいかを判定することが目的である。

軽微な品質改善、可読性向上、将来的なリファクタリング提案があるだけでは PR を止めない。
そのような問題は、レビューコメントとして残しつつ `mergeable` とする。

## `human-review` とする条件

次のいずれかに該当する場合は `human-review` とする。

1. `risk-map.yaml` で `high` と定義された領域の変更が含まれる
2. `human-review-required.md` に定義された条件に合致すると判断できる
3. デプロイすると一般的な操作で即不具合につながる可能性が高い
4. 認証、認可、IDOR、情報漏洩などの重大なセキュリティ問題が残る
5. 深刻なパフォーマンス劣化が予測される

## `mergeable` とする条件

次をすべて満たす場合は `mergeable` とする。

- `human-review` 条件に該当しない
- blocking な問題が確認できない
- 残っている指摘が、別 PR で安全に対処可能な範囲に留まる

## Non-blocking の例

次のような問題は `mergeable` のままコメントで指摘する。

- 命名の改善
- 可読性の改善
- 軽微な構造改善
- リファクタリングの提案
- テストの追加提案
- 将来的な保守性改善

non-blocking な指摘は、別 PR での対応も可能であることをコメント内に明示することが望ましい。

## 注意点

- repository に存在しないルールを新たに作らない
- `human-review` は安全上の懸念がある時だけ使う
- 判定に迷う場合でも、軽微な品質論点だけなら `mergeable` とする

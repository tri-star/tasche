# プロジェクト概要関連ドキュメント

## プロジェクト概要

ユーザーの入力する project-overview を元に、プロジェクト全体の概要をまとめたドキュメント(`<project-root>/docs/project-overview.md`)を作成します。
(`<project-root>` は .gitフォルダがあるプロジェクトのルートと見なせるフォルダ)
Plan agentを利用して、必要な情報が漏れなく含まれるようにします。

### プロジェクト概要の内容

テンプレート: `../templates/project-overview.md`

このドキュメントには最終的に以下がまとめられている必要があります。

- プロジェクトの目的・背景
  - 今後のタスクに取り組む際全体像を意識しながら作業できるようにするため、プロジェクトの目的や背景をまとめたドキュメントを作成します。
    **"このプロジェクトはどんな製品で、何を解決することを目指しているか"の情報は必ず必要**
- プロジェクトの技術スタック
  - プロジェクトフォルダにほとんどファイルがない状態の場合はユーザーと相談し技術スタックを決定します。
  - プロジェクトに既にソースコードが存在する場合は Explore Agent を利用しコードベースを調査し、技術スタックを決定します。
- backendのフォルダ構造・命名規則ドキュメントへのリンク
  - `<project-root>/packages/frontend/docs/folder-structure.md`
  - `<project-root>/packages/frontend/docs/naming-convention.md`
  - `<project-root>/packages/backend/docs/folder-structure.md`
  - `<project-root>/packages/backend/docs/naming-convention.md`

### ドキュメント作成時の注意事項

- ユーザーの入力からは判断できない、または懸念がある事項がある場合は、ドキュメントを作成する前にユーザーに確認を求めてください。
  - その際、推奨事項がある場合は推奨事項の提案と、その理由も合わせて伝えるようにしてください。

- ドキュメント作成後はユーザーのレビューを求め、OKが出るまでは次のステップに進まないようにしてください。

## 2. README.md

現時点での内容で、 `<project-root>/README.md` を作成します。
これには以下のような見出しと内容を含めます。

- プロジェクトの概要
- ドキュメントへのリンク
  - プロジェクト概要
  - ADR (この時点では未作成だが、TBDなどでセクションだけ設けておく)
- 技術スタック
  - 使用言語やフレームワーク、ライブラリ、CI/CDツールなどを記載。未定のものは"TBD"として記載しておく。
- アーキテクチャ
  - デプロイ後のシステムのアーキテクチャ。未定義の場合、TBDとして記載しておく。

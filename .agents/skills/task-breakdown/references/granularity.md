# 細分化の粒度ガイドライン

## 基本原則

- 1タスク = 1機能単位
- 1機能には複数画面・複数APIが含まれうる

## 粒度の判断基準

### 横展開可能な場合（粗い粒度）

プロジェクトに既存の類似実装がある場合、パターンをまとめて1タスクにできる。

例: 既にCRUD画面が存在するプロジェクトで新エンティティのCRUDを追加
→ 「○○管理画面（CRUD）」で1タスク

### 横展開不可の場合（細かい粒度）

プロジェクトに前例がない機能は、個別タスクに分割する。

例: プロジェクト初のCRUD機能を実装する場合
→ 「○○の一覧表示（Read）」「○○の新規作成（Create）」「○○の編集（Update）」「○○の削除（Delete）」で各1タスク

### その他の分割例

| 機能種別 | 粒度の目安 |
|----------|-----------|
| バッチ処理 | 1バッチ = 1タスク |
| 画面（CRUD、横展開可） | 1画面 = 1タスク |
| 画面（CRUD、横展開不可） | C/R/U/D各1タスク |
| 共通基盤（認証、DB接続等） | 1基盤機能 = 1タスク |
| 設定・環境構築 | 1環境 = 1タスク |

## 禁止事項

- backend/frontendでタスクを分割しない（1画面の実装はまとめて1タスク）
- 1タスクを「設計」「実装」「テスト」のフェーズで分割しない

# タスクの目標

- 1. ユーザーへのヒアリング、MVPの定義
- 2. mvp.mdの作成
- 3. マイルストーンの定義
- 4. Planeにおおまかな粒度のタスクを作成

# フォルダ

- `<project-root>`: .gitフォルダがあるプロジェクトのルートと見なせるフォルダ

# MVP定義にあたっての条件

- 開発環境・本番環境のうち、開発環境にデプロイ出来ている状態を目指します。

# 作業詳細

## 1. ユーザーへのヒアリング、MVPの定義

MVPの定義過程でユーザーの要求をヒアリングして、必要に応じ `project-overview.md` を更新

## 2. mvp.mdの作成

ユーザーへのヒアリング結果をもとに、MVPの内容を以下のファイルにまとめる

- `<project-root>/docs/mvp.md` を作成

## 3. マイルストーンの定義

MVPを達成するまでのマイルストーンを定義します。

### マイルストーンの定義のポイント

次のような点を重視してマイルストーンの順番を決定、定義していきます。

- エンドユーザーになるべく早く動くものを見せながら進められるようにする
  - NG例1： 先に認証基盤から着手する
    - 目指したい方向： アプリケーションとして重要な機能に先に着手、認証はそれよりも低い優先度で着手し、後から組み込む
- 先に整備しておくことで大きな恩恵を受けられる要素は優先度を上げて取り組む
  - 例： CI/CD, ローカルでのLint/Formatの自動化、デザイントークンの策定
    (技術スタックの決定を後回しにした場合、この部分もスキップする)
- `デザイントークン策定` を、frontendのページやコンポーネント作成よりも早い段階で着手する

## 4. Planeにおおまかな粒度のタスクを作成

Plane上にMVP完了までの具体的な一式と、MVP以降のおおまかなタスク(粒度大)を作成します。

plane-project-manager Agent を利用して各マイルストーンを"Module"として登録していきます。
"Module"内にはまず、"タスク細分化"というタイトルで1つだけタスク(WorkItem)を作成するようにします。
(マイルストーンに着手する時、最初にタスクの細分化を行う想定)

"タスク細分化"タスクの内容には、マイルストーンの概要と、どんなタスクが含まれるかの情報を記載しておきます。

# 概要

あなたは Plane 上のタスクのトリアージを行う担当です。

ユーザーからの要望に応じ、`plane-project-manager` エージェントを利用して以下のような作業を行います。

- Plane プロジェクトの特定
- 次に着手するべきタスクの選択
- タスクの状態更新

# Skillで利用するフォルダ

- `<project-root>`: `.claude` フォルダのある、プロジェクトのルートと見なせるフォルダ。

# Plane プロジェクトの特定

- 最初にPlaneのプロジェクトIDを特定します。
  - これまでの会話やメモリーからPlaneのプロジェクトIDを特定できない場合、まずユーザーに尋ねてください。
    - ユーザーから回答を受け取ったら、 `<project-root>/.claude/agent-memory/plane-project-manager/MEMORY.md` にプロジェクトIDの情報を追記してください。
  - **プロジェクトIDを特定できないと他プロジェクトのタスクを誤って選択してしまう可能性があるので、必ず特定してください。**

# 次に着手するべきタスクの選択

- "Review"状態のタスクがある場合
  - タスクの本文やコメントなどからPRを特定し、PRの状態を確認します。
  - PRがマージされている場合、タスクを"Done"に更新して先に進みます。
  - PRがマージされていない場合、ユーザーに「`<task-id>` が現在レビュー待ち」と報告し、次に着手するタスクの選択に進みます。
- "In Progress"状態のタスクがある場合
  - タスクの内容をコメントも含めて確認し、ユーザーに「`<task-id>` が現在着手中になっていて、○○の状態になっている」と報告します。
  - 複数存在する場合は全て確認した上で、最も優先度が高いと思われるものをユーザーに提案します。
    - **ユーザーから「自動的に進めて欲しい」と要望があった場合は提案せず、`task-breakdown`, `task-workflow` スキルでそのままタスクに着手します**
- "In Progress"状態のタスクがない場合
  - "to-do","backlog"の中から優先度の高いものを選択します。
    判断には、各タスクが属しているモジュールやタスク自体の優先度、タスクの内容を用いて判断を行います。
  - 選択したタスクのうち高優先度と思われるものをピックアップしてユーザーに提案します。
    - **ユーザーから「自動的に進めて欲しい」と要望があった場合は提案せず、`task-breakdown`, `task-workflow` スキルでそのままタスクに着手します**

# タスクの状態更新

- 着手することが決まったタスクは `In Progress` 状態に更新します。
